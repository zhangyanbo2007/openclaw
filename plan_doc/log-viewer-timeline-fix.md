# 日志查看器时间轴显示修复

## 问题描述

用户反馈日志查看器的会话时间轴显示存在以下问题：

### 1. 序号跳跃问题
- **现象**：第 1 轮后直接跳到第 3 轮
- **原因**：代码将用户消息、工具调用、工具响应、模型响应各作为独立事件，每个事件独立递增序号

### 2. 上下文历史问题
- **现象**：模型输入中包含了完整的对话历史（如"你好"），但看起来像是新的输入
- **原因**：每个请求的 `messages` 数组包含累积的对话历史，但在时间轴上没有清晰标示这是历史上下文

### 3. 工具调用时序问题
- **现象**：没有看到工具调用过程，直接显示结果
- **原因**：工具调用和响应在同一个请求中处理，没有体现真实时序

### 4. 缺少模型调用步骤
- **现象**：用户问"查天气"后，直接显示模型输出天气，中间缺少"模型决定调用工具"的步骤
- **原因**：日志数据中 `tool_calls` 字段为空，代理服务器转发时未保留

## 解决方案

### 最终方案：单一模型角色

**简化为只显示"模型"角色**，按时间顺序罗列每次模型调用的输入和输出：

1. **Model 行**：显示模型的完整输入和输出
   - 输入：`messages` 数组（包含 system/user/tool 角色）
   - 输出：`content` + `reasoning` + `tool_calls`
   - 信息：代理 URL、原始 URL、模型名称、token 使用
   - 轮次号：只在用户消息变化时显示

**优势：**
- 简洁明了，没有复杂的角色区分
- 适用于任何智能体的模型日志
- 完整保留模型调用的所有信息

**数据结构：**

```python
timeline_events = [
    {"round": 1, "role": "model", "input": [...], "output": "...", "model_info": {...}},
    {"round": 2, "role": "model", "input": [...], "output": "...", "model_info": {...}},
]
```
