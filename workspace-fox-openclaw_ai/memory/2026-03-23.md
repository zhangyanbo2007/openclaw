## 学习时间
2026-03-23

## 核心架构

### Gateway 架构
- **单进程**：一个 Gateway 进程支持所有功能
- **WebSocket Server**：所有连接通过 WS (端口 18789)
- **Clients**：macOS App、CLI、Web Admin (role: "operator")
- **Nodes**：macOS/iOS/Android 设备 (role: "node" + caps)
- **Channels**：WhatsApp、Telegram、Discord、Slack、Signal、iMessage、飞书等

### Session 管理
- **会话键格式**：
  - 私聊：`agent:<id>:main` 或 `agent:<id>:<channel>:direct:<peerId>`
  - 群聊：`agent:<id>:<channel>:group:<id>`
- **dmScope**：
  - `main` (默认)：所有私聊共享一个会话
  - `per-channel-peer` (推荐)：按渠道 + 发送者隔离
  - `per-account-channel-peer`：按账户 + 渠道 + 发送者隔离
- **身份链接**：`session.identityLinks` 合并同一人在多渠道的会话
- **安全私聊模式**：多用户时启用 `dmScope: "per-channel-peer"` 防止隐私泄露

### 上下文加载差异
- **私聊**：✅ 加载 MEMORY.md（长期记忆）
- **群聊**：❌ 不加载 MEMORY.md（防止隐私泄露）
- **其他文件**：SOUL.md、USER.md、AGENTS.md、TOOLS.md 都加载

## Agent Loop（智能体循环）

### 核心流程
1. **Context Assembly**：上下文组装（系统提示 + bootstrap 文件）
2. **Model Inference**：LLM 推理
3. **Tool Execution**：工具执行
4. **Turn End Check**：轮次结束检查

### sessions_spawn 工具
- **工具描述**：只有 "Spawn a background sub-agent" 一句话
- **官方 AGENTS.md**：❌ 没有使用场景说明
- **模型决策依据**：
  - ✅ 工具名称语义（"spawn" 暗示后台）
  - ✅ 参数结构（task 参数暗示独立任务）
  - ✅ 模型训练数据中的通用模式
  - ❌ 框架没有明确引导
- **用户自定义**：可在 TOOLS.md 中添加使用指南

## 子智能体（Sub-Agent）

### 核心特性
- **独立会话**：`agent:<agentId>:subagent:<uuid>`
- **后台运行**：非阻塞，立即返回 runId
- **结果通知**：通过 Announce 机制发回请求者频道
- **工具限制**：默认没有 session 工具
- **自动归档**：完成后 60 分钟自动归档

### 嵌套支持
- **maxSpawnDepth: 1**（默认）：Main → Sub-Agent (Leaf)
- **maxSpawnDepth: 2**（Orchestrator 模式）：Main → Sub-Agent → Sub-Sub-Agent

### 并行执行
- ✅ 支持并行
- **全局并发限制**：`maxConcurrent: 8`（默认）
- **每 Agent 最大子节点**：`maxChildrenPerAgent: 5`

### 与 Claude Code 对比
| 特性 | OpenClaw | Claude Code |
|------|----------|-------------|
| 定位 | 多渠道 Gateway + 多智能体编排 | 终端编码助手 |
| 子智能体 | 后台异步任务 | 并行研究/任务执行 |
| 嵌套 | 可配置（maxSpawnDepth: 1-2） | 支持（plan/execute 模式） |
| 并行 | 全局并发限制 | 多任务并行 |
| 渠道 | WhatsApp/Telegram/Discord 等 | 终端/IDE 插件 |

## Context Engine（上下文引擎）

### 核心职责
控制如何为每次模型运行构建上下文

### 四个生命周期阶段
1. **Ingest**：新消息添加到会话时存储/索引
2. **Assemble**：模型运行前返回有序消息集（符合 token 预算）
3. **Compact**：上下文窗口满时总结旧历史
4. **After Turn**：运行完成后持久化状态

### 插件系统
- **Legacy 引擎**（默认）：pass-through，使用内置压缩
- **插件引擎**：可实现自定义策略（DAG 总结、向量检索等）
- **ownsCompaction**：
  - `true`：引擎接管压缩，禁用 Pi 内置自动压缩
  - `false`：Pi 内置压缩仍可能运行

## Memory（记忆系统）

### 两层结构
- **memory/YYYY-MM-DD.md**：日常日志（追加式，读今天 + 昨天）
- **MEMORY.md**：长期记忆（仅主会话加载）

### 记忆工具
- **memory_search**：语义检索索引片段（由 memory 插件提供）
- **memory_get**：精准读取特定文件/行范围

### 工具对比
| 工具 | 用途 | 输入 | 输出 |
|------|------|------|------|
| memory_get | 精准读取 | 文件路径 + 行号 | 文件原文 |
| memory_search | 语义检索 | 查询关键词 | 相关片段列表 |

### 向量搜索（可选）
- 支持嵌入提供商：OpenAI、Gemini、Voyage、Mistral、Ollama、GGUF
- 混合搜索：BM25 + 向量
- MMR 重排序、时间衰减

## Compaction（压缩）

### 核心机制
- **目的**：总结旧历史为紧凑摘要，保留最近消息
- **持久化**：压缩摘要写入 JSONL 历史
- **触发**：接近上下文窗口时自动运行

### 配置
```json5
{
  agents: {
    defaults: {
      compaction: {
        model: "openrouter/anthropic/claude-sonnet-4-6",  // 专用压缩模型
        identifierPolicy: "strict",  // 保护标识符
        reserveTokensFloor: 20000,
      },
    },
  },
}
```

### 压缩前记忆刷新
- 静默轮次提醒模型写入持久记忆
- 通常回复 NO_REPLY（用户看不到）

## Session Pruning（会话修剪）

### 与 Compaction 的区别
| 特性 | Session Pruning | Compaction |
|------|-----------------|------------|
| 范围 | 仅工具结果 | 所有消息 |
| 持久化 | ❌ 仅内存 | ✅ 写入 JSONL |
| 触发 | 每次 LLM 调用前（TTL 过期后） | 接近上下文窗口 |
| 目的 | 优化缓存成本 | 节省上下文空间 |

### 触发条件
- `mode: "cache-ttl"` 启用
- 上次 Anthropic 调用 > TTL（默认 5 分钟）
- 仅适用于 Anthropic API 调用

### 修剪方式
- **Soft-Trim**：保留头尾，中间插入 "..."
- **Hard-Clear**：替换为占位符 `[Old tool result content cleared]`

## Skills（技能系统）

### 三层来源
1. **Bundled skills**：npm 包/OpenClaw.app 内（优先级最低）
2. **Managed/Local skills**：`~/.openclaw/skills`（中等）
3. **Workspace skills**：`<workspace>/skills`（最高）

### ClawHub
- 公共技能注册表：https://clawhub.com
- 安装：`openclaw skills install <skill-slug>`
- 更新：`openclaw skills update --all`

### SKILL.md 格式
```markdown
---
name: skill-name
description: 简短描述
metadata: {"openclaw": {"requires": {"bins": ["xx"], "env": ["API_KEY"]}}}
---

# Skill Instructions
使用说明...
```

### Gating（加载时过滤）
- `requires.bins`：二进制文件必须存在
- `requires.env`：环境变量必须存在
- `requires.config`：配置项必须为 truthy
- `os`：平台限制

### Token 成本
- 基础开销：195 字符（≥1 技能时）
- 每技能：97 字符 + 字段长度
- 估算：~4 字符/token，97 字符 ≈ 24 tokens/技能

## 配置要点

### Memory 插件启用
```json5
{
  plugins: {
    allow: ["openclaw-lark", "memory-core"],
    slots: {
      memory: "memory-core",
    },
    entries: {
      "memory-core": { enabled: true },
    },
  },
}
```

### 会话维护
```json5
{
  session: {
    maintenance: {
      mode: "enforce",
      pruneAfter: "30d",
      maxEntries: 500,
      rotateBytes: "10mb",
    },
  },
}
```

## 关键决策总结

### 何时使用 sessions_spawn
- 预计耗时 >5 分钟的任务
- 研究/调研类任务
- 可并行化的多个子任务
- 需要使用不同模型的任务

### 何时直接调用工具
- 简单查询（read/exec）
- 预计耗时 <1 分钟
- 需要立即结果继续对话

### 安全私聊模式
- 多用户时启用 `dmScope: "per-channel-peer"`
- 防止个人隐私泄露到群聊
