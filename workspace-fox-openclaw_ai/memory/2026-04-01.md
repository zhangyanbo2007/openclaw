### 2. OpenClaw Memory 向量检索配置

**问题诊断：**
```bash
openclaw memory status
```
输出显示：
- `Provider: none (requested: auto)` — 未配置嵌入模型
- `Vector: unknown` — 向量检索未启用
- `Indexed: 0/X files` — 文件未索引
- `FTS: ready` — 仅全文搜索可用

**配置步骤：**

1. 在 `~/.openclaw/openclaw.json` 中添加：
```json
"memorySearch": {
  "sources": ["memory"],
  "provider": "custom-dashscope-aliyuncs-com",
  "model": "text-embedding-v3"
}
```

2. 验证配置：
```bash
openclaw config validate
```

3. 重启 gateway：
```bash
systemctl --user restart openclaw-gateway
```

4. 重建索引：
```bash
openclaw memory index --force
```

**嵌入模型选择：**
- 阿里云 DashScope: `text-embedding-v3`
- 需要配置 API key（通过 auth profile 或环境变量）

### 3. OpenClaw 配置结构要点

**配置文件位置：** `~/.openclaw/openclaw.json`

**关键配置项：**
- `auth.profiles` — 定义 provider 认证模式
- `models.providers` — 定义模型列表和参数
- `memorySearch` — Memory 向量检索配置
- `plugins.entries` — 插件配置
- `agents.list` — Agent 列表和绑定

**注意事项：**
- `apiKey` 不能直接放在 `auth.profiles` 中（会验证失败）
- 嵌入模型需要单独配置 `memorySearch` 段落
- 配置修改后需要重启 gateway 生效

---

### 5. OpenClaw 批准机制

**批准格式：**
```
/approve <short-id> allow-once    # 批准一次
/approve <short-id> allow-always  # 永久批准同类命令
```

**建议：** 对常用命令使用 `allow-always` 减少频繁批准
